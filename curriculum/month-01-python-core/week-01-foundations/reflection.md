## What I built

- What working thing did I build this week?
- Which command proves it works?
- Which file best represents the core of my Week 1 work?
- What part of the project now feels more real than it did before this chapter?

## What I learned

- What is the difference between a repository, a package, and a module?
- What is a virtual environment doing for me?
- What does editable install mean in my own words?
- Why does this project use `src/` layout?
- What does the CLI entry point connect together?

## What broke

- What failed first when I tried to run the project?
- What command triggered the failure?
- What was the exact error or symptom?
- Which layer was broken: install, import, CLI, utility, or test?

## What I misunderstood

- Which concept did I think I understood but actually did not?
- What incorrect mental model was I carrying?
- What corrected that misunderstanding?
- What evidence now makes the concept clearer?

## How I fixed it

- What specific steps did I take to diagnose the problem?
- Which command gave the most useful information?
- Which file did I inspect?
- What was the real fix?
- What proof convinced me the fix actually worked?

## What tests helped

- Which unit test gave me the clearest signal?
- Which CLI test gave me the clearest signal?
- What behavior did those tests protect?
- What did the tests teach me about function design?

## What tests were missing

- Which edge case do I think deserves a test that is not here yet?
- Why would that test matter?
- Would it be a unit test or an end-to-end test?
- What behavior would it pin down?

## What I overengineered

- Did I make any part more complicated than the week required?
- Where did I add abstraction before earning it?
- What would a simpler version have looked like?
- Why was simplicity the better choice here?

## What I underengineered

- Where was my design too loose or too fragile?
- Which validation, explanation, or test did I skip at first?
- What weak spot became visible because of that?
- What would make it more robust next time?

## What I can explain aloud

- Can I explain how `researchops --help` becomes a Python object being executed?
- Can I explain why `find_pdfs()` returns data instead of printing directly?
- Can I explain why `tmp_path` is used in tests?
- Can I explain why deterministic ordering matters?
- Can I explain why this week is architecture work, not just setup work?

## What still feels weak

- Which command still feels magical?
- Which file still feels dense or confusing?
- Which term do I still hesitate over?
- What do I need to practice again before moving on?

## Confidence score

- What is my Week 1 confidence score from 1 to 10?
- Why did I choose that score?
- What would need to become true for the score to go up by one point?

## Whether I am allowed to move forward

- Can I run the validation commands without guessing?
- Can I answer the ruthless mentor questions honestly?
- Can I explain the Week 1 execution flow from terminal to output?
- Am I moving forward because I understand, or because I am impatient?
- Based on my evidence, am I actually allowed to move to Week 2?
