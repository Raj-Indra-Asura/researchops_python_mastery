# Week 19 Break-It: Documentation

## Experiment 1: The Fresh Clone Test
Clone your repository to a new directory and follow your own README from scratch. What breaks? What's unclear?

## Experiment 2: The 5-Minute Test
Give your README to someone else (or pretend to). Can they understand what the project does in 5 minutes without asking you any questions?

## Experiment 3: The Diagram Accuracy Test
Draw the architecture diagram from memory, then compare to the actual code. Are there modules you forgot? Are there dependencies you got wrong?

## Experiment 4: The Stale Documentation Test
Find three places where your documentation says something that no longer matches the code. Fix them.

## What Did You Learn?
- How long did the fresh clone test take?
- Which part of the README was most confusing?
- What was missing from your architecture diagram?
