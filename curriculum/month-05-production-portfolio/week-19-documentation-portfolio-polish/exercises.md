# Week 19 Exercises: Documentation and Portfolio Polish

## Warm-Up Exercises

1. Find three open-source Python projects on GitHub with excellent READMEs. List what they have in common.
2. Write one paragraph explaining ResearchOps to someone who has never heard of it.
3. Draw (on paper) the module dependency diagram from memory. Then compare to the code.

## Project Exercises

1. **Write the final README.** Follow the structure in notes.md. Spend at least 90 minutes on it.
2. **Create a Mermaid diagram** for the ingestion data flow: directory → find_pdfs → parse_pdf → IngestionService → SQLitePaperRepository.
3. **Create a Mermaid diagram** for the RAG pipeline: question → embed → cosine_search → prompt → LLM → answer.
4. **Write `docs/demo.md`**: a script for a 5-minute live demo. Include exact commands and expected output.
5. **Write `docs/retrospective.md`**: 19 weeks of learning in one document. What changed in your understanding?

## Stretch Exercises

1. Record a screen capture demo (Loom, OBS, or similar). Link it from the README.
2. Create a GitHub Pages site from the docs directory.
3. Write a blog post (even a draft) about building ResearchOps.

## Writing Questions

Answer these in your `reflection.md`:
1. What is the most important design decision you made in this project? Why?
2. What would you do differently if you started over?
3. What is the hardest concept you encountered in 19 weeks?
4. What are you most proud of?
5. What would you build in the next 5 weeks if you continued?
