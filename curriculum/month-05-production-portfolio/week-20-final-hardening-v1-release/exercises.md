# Week 20 Exercises: Final Hardening and v1.0 Release

## Warm-Up Exercises

1. Write a definition of "done" for a software project in your own words.
2. List 5 things you would want to add to ResearchOps in v1.1. Now resist the urge to add them to v1.0.
3. Explain semantic versioning (MAJOR.MINOR.PATCH) in one paragraph without referencing the notes.

## Project Exercises

1. **Run the full pre-release checklist** from notes.md. Fix anything that fails.
2. **Complete `CHANGELOG.md`** with entries for all 20 weeks.
3. **Bump the version** to `1.0.0` in `pyproject.toml` and `src/researchops/__init__.py`.
4. **Run `docs/demo.md`** from start to finish. Fix any commands that don't work.
5. **Create the `v1.0.0` tag** and push it to GitHub.

## Stretch Exercises

1. Create a GitHub Release with release notes summarizing what v1.0.0 includes.
2. Write a LinkedIn post about completing the project.
3. Create a `ROADMAP.md` entry for what v1.1 would include.

## Writing Questions

Answer these in your final `reflection.md`:
1. What is the most important thing you learned in 20 weeks?
2. How is your understanding of Python different now vs. when you started?
3. What would you tell someone starting this curriculum?
4. Rate your confidence in: Python fundamentals, clean architecture, ML engineering, AI engineering.
5. What's next?
