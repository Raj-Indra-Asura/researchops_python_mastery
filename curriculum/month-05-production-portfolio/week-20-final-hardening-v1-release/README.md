# Week 20: Final Hardening and v1.0 Release

## Overview
This is the last week. The goal is not to add features — it's to finish. Scope control is itself a skill. Your job this week is to make everything that exists work perfectly, then tag a v1.0.0 release.

## Learning Objectives
1. Understand what "done" means for a software project
2. Complete a systematic pre-release checklist
3. Write a changelog that tells the story of the project
4. Tag a release and understand semantic versioning
5. Practice the discipline of stopping

## Project Milestone
> **`v1.0.0` is tagged. CI is green. A demo exists. The project is portfolio-ready.**

## Files to Create/Update
- `CHANGELOG.md` — complete version history
- `ROADMAP.md` — update all week statuses to ✅
- `pyproject.toml` — bump version to `1.0.0`
- `src/researchops/__init__.py` — update `__version__`

## Concepts Covered
- Release discipline
- Semantic versioning (MAJOR.MINOR.PATCH)
- Changelog writing
- Scope control — finishing vs. polishing forever
- Retrospective — what did 20 weeks teach you?

## Expected Deliverables
- [ ] All CI checks green
- [ ] `CHANGELOG.md` complete
- [ ] Version bumped to `1.0.0`
- [ ] `v1.0.0` git tag created
- [ ] Final `reflection.md` written
- [ ] ROADMAP.md shows all weeks complete

## Definition of Done
- [ ] `pytest` — 0 failures
- [ ] `ruff check src tests` — 0 errors
- [ ] `researchops --help` works
- [ ] Full demo from `docs/demo.md` runs without errors
- [ ] `git tag v1.0.0` exists
- [ ] You would show this to an interviewer without embarrassment
